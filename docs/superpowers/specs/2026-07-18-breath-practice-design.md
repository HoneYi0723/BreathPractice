# 呼吸練習 App 設計規格

**日期**：2026-07-18
**狀態**：設計定案，待實作

## 目標

一個介面簡單的呼吸練習網頁，可在 Android 手機上執行，用音檔引導使用者完成「開始 → 吸氣 → 閉氣 → 呼氣 → 閉氣 →（重複）→ 結束」。做一次、部署一次，透過可分享網址讓自己與他人使用。

## 交付與部署方向

- **純網頁**：HTML + CSS + JavaScript，不依賴任何框架，方便部署與離線。
- **部署**：整包（含 `sound/`）上傳到靜態空間（GitHub Pages / Netlify / Cloudflare Pages），得到一個公開、可分享的網址。
- **PWA**：附 `manifest.json` 與 Service Worker，可「加到主畫面」全螢幕執行；五個音檔列入快取，**離線亦可使用**。
- **記憶設定**：以瀏覽器 `localStorage` 記住上次的秒數/時間設定，下次開啟自動沿用。

### 檔案結構

```
BreathPractice/
├── index.html
├── app.js
├── style.css
├── manifest.json
├── service-worker.js
└── sound/
    ├── Start.mp3    # 練習開始
    ├── in.mp3       # 吸氣提示
    ├── hold.mp3     # 閉氣提示
    ├── out.mp3      # 呼氣提示
    └── finish.mp3   # 練習結束
```

## 畫面一：設定畫面（直向六列）

| 列 | 內容 | 調整方式 | 預設值 |
|---|---|---|---|
| 1 | 吸氣秒數 | − / + 按鈕 | 2 |
| 2 | 閉氣秒數（吸氣後） | − / + 按鈕，可設 0 | 2 |
| 3 | 呼氣秒數 | − / + 按鈕 | 2 |
| 4 | 閉氣秒數（呼氣後） | − / + 按鈕，可設 0 | 2 |
| 5 | 練習時間（分鐘） | − / + 按鈕 | 5 |
| 6 | **開始** 按鈕 | 大按鈕 | — |

- 輸入方式一律採 **− / +** 按鈕：手機好點、不會彈出鍵盤遮畫面。
- 秒數欄位下限：吸氣/呼氣至少 1 秒；兩個閉氣欄位可低到 0。
- 設定畫面**不放離開按鈕**（網頁無法真正關閉自己）；中途停止改由練習畫面處理。

## 畫面二：練習畫面（全螢幕專屬）

按「開始」後切換到此畫面，顯示：

- **目前階段名稱**（吸氣 / 閉氣 / 呼氣 / 閉氣）
- **該階段倒數秒數**
- **剩餘總時間**
- **隨呼吸放大／縮小的圓圈動畫**：吸氣時圓圈放大、呼氣時縮小、閉氣時停住，配合當前階段
- **停止** 按鈕

## 流程

1. 按「開始」→ 播 `Start.mp3` → 進入循環。
2. 每一輪循環依序：
   - 吸氣：播 `in.mp3`，倒數吸氣秒數，圓圈放大
   - 閉氣（吸氣後）：播 `hold.mp3`，倒數，圓圈停住
   - 呼氣：播 `out.mp3`，倒數，圓圈縮小
   - 閉氣（呼氣後）：播 `hold.mp3`，倒數，圓圈停住
3. **秒數為 0 的閉氣階段自動跳過**（不播音、不倒數，直接進入下一階段）。同一個 App 因此既能做箱式呼吸（4-4-4-4），也能做單純的吸-呼。
4. **練習時間到**：不立即中斷，先做完**當前整輪循環**，再播 `finish.mp3`，然後回到設定畫面。
5. **按「停止」**：立即中止，回到設定畫面。

## 行為決策紀錄

- 練習中切換到專屬全螢幕畫面（非留在設定頁）。
- 數值調整用 − / + 按鈕。
- 「練習時間」為總分鐘數；到時完成當前循環後才結束。
- 閉氣可設 0 秒並自動跳過。
- 設定畫面無離開鈕；練習中提供「停止」。
- 練習畫面加圓圈呼吸動畫。

## 技術注意事項

- **首次播放需使用者手勢**：Android 瀏覽器禁止自動播放音訊，第一段音效必須在使用者按下「開始」後觸發；建議在按下開始時一併「解鎖」所有音檔（預先 load / 靜音播放一次）以確保後續各階段能順暢播放。
- **不可用 `file://` 直開**：本機測試需透過簡易伺服器（如 `npx serve`）或部署後以網址開啟，否則部分手機瀏覽器載入音檔會失敗。
- **音檔與倒數的時間關係**：各階段開始時觸發對應音效，接著倒數該階段秒數；音檔本身較短（in/out/hold 約 1–2 秒），在秒數內播完即可。
